# Plan 04-01: HTTP Client Transport

## Goal
Add `connect_http(url)` to McpClient so TL can connect to remote MCP servers over HTTP+SSE.

## Tasks

### Task 1: Add `connect_http()` to McpClient
File: `crates/tl-mcp/src/client.rs`

Add method using rmcp's `StreamableHttpClientTransport`:
```rust
pub fn connect_http(url: &str, security_policy: Option<&SecurityPolicy>) -> Result<Self, McpError>
pub fn connect_http_with_runtime(url: &str, security_policy: Option<&SecurityPolicy>, runtime: Arc<Runtime>) -> Result<Self, McpError>
```
- Check SecurityPolicy (no subprocess gating needed — this is network)
- Create `StreamableHttpClientTransport::<reqwest::Client>::from_uri(url)`
- Call `TlClientHandler.serve(transport).await` — same handshake as stdio
- Return McpClient (struct is transport-agnostic)

### Task 2: URL auto-detection in VM McpConnect
File: `crates/tl-compiler/src/vm.rs`

In `BuiltinId::McpConnect` handler: if command starts with `http://` or `https://`, call `connect_http()` instead of `connect()`.

### Task 3: URL auto-detection in Interpreter mcp_connect
File: `crates/tl-interpreter/src/lib.rs`

Same logic as Task 2 in interpreter's `"mcp_connect"` arm.

## Verification
