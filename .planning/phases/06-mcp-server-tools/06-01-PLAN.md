# Plan 06-01: Channel-Based Server Dispatch Infrastructure

## Goal
Add channel-based tool dispatch to tl-mcp server so VM/interpreter can process MCP tool calls on the main thread.

## Architecture
```
Server Thread (tokio)         Main Thread (VM)
─────────────────────         ──────────────────
Tool handler receives call → send(request_tx, (name, args))
                             ← recv(request_rx) → call TL function
recv(response_rx) ← result  ← send(response_tx, result)
```

## Tasks

### Task 1: Add ToolCallRequest type and channel builder
File: `crates/tl-mcp/src/server.rs`

```rust
pub struct ToolCallRequest {
    pub tool_name: String,
    pub arguments: serde_json::Value,
    pub response_tx: std::sync::mpsc::Sender<Result<serde_json::Value, String>>,
}

pub struct ChannelToolDef {
    pub name: String,
    pub description: String,
    pub input_schema: serde_json::Value,
}
```

Add `TlServerBuilder::channel_tool()` that registers tools backed by mpsc channels instead of closures.

### Task 2: serve_stdio_background()
File: `crates/tl-mcp/src/server.rs`

```rust
pub fn serve_stdio_background(handler: TlServerHandler)
    -> Result<(JoinHandle<Result<(), McpError>>, mpsc::Receiver<ToolCallRequest>), McpError>
```

Spawns server on background thread, returns handle + request receiver.

### Task 3: Unit tests
Test channel flow: build handler with channel tools, simulate tool call, verify request/response roundtrip.
