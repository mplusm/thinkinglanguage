# Plan 06-02: VM & Interpreter McpServe Implementation

## Goal
Implement the McpServe builtin (BuiltinId 220) in both VM and interpreter.

## TL Syntax
```
fn greet(name) {
    return "Hello, " + name + "!"
}

mcp_serve([
    map_from("name", "greet", "description", "Greet someone", "handler", greet)
])
```

## Tasks

### Task 1: VM McpServe dispatch
File: `crates/tl-compiler/src/vm.rs`

Replace stub with:
1. Parse args[0] as list of tool definition maps
2. Extract name, description, handler (VmValue::Function), input_schema from each
3. Build channel-based TlServerHandler via ChannelToolDefs
4. Spawn server via serve_stdio_background()
5. Enter blocking dispatch loop:
   - recv ToolCallRequest
   - Find matching TL function
   - Convert JSON args to VmValue args
   - Call function via call_value()
   - Send result back via response channel
6. Return VmValue::None when channel closes

### Task 2: Interpreter mcp_serve dispatch
File: `crates/tl-interpreter/src/lib.rs`

Mirror Task 1 for interpreter. Use call_function_value() for dispatch.

### Task 3: json_to_vmvalue helper
File: `crates/tl-compiler/src/vm.rs`

Add helper to convert serde_json::Value → Vec<VmValue> for function call args.
