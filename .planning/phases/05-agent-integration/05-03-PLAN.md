# Plan 05-03: agent_add_mcp Builtin & Tests

## Goal
Programmatic API for adding MCP tools to agents, plus integration tests.

## Tasks

### Task 1: agent_add_mcp builtin (BuiltinId 223)
Files: `crates/tl-compiler/src/chunk.rs`, `crates/tl-compiler/src/vm.rs`, `crates/tl-interpreter/src/lib.rs`

`agent_add_mcp(agent, mcp_client)` → new agent with MCP tools merged in:
- Extract AgentDef from agent value
- Query list_tools() from client
- Convert MCP Tools to AgentTools
- Store client in VM-level side table keyed by agent name
- Return new AgentDef with merged tools

### Task 2: Multi-server namespace conflict resolution
In exec_agent_loop (both backends):
- Detect conflicting tool names across MCP servers
- Prefix with server name: `{server_name}__{tool_name}`
- Log warning when prefixing occurs

### Task 3: Integration tests
File: `tests/mcp_agent.tl`

Test scenarios:
1. Agent with `mcp_servers: [client]` auto-discovers and dispatches
2. `agent_add_mcp` programmatic API
3. MCP tool call error handling
4. Lifecycle hooks fire for MCP tool calls
