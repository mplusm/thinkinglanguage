# Plan 05-02: Agent Loop MCP Tool Discovery & Dispatch

## Goal
Modify exec_agent_loop in both VM and interpreter to discover MCP tools and dispatch MCP tool calls.

## Tasks

### Task 1: VM exec_agent_loop MCP support
File: `crates/tl-compiler/src/vm.rs`

Extend `exec_agent_loop` signature with `mcp_clients: &[Arc<McpClient>]`:
1. **Tool discovery**: Call `list_tools()` on each client, convert rmcp Tool to OpenAI tool JSON format, append to tools_json
2. **Dispatch map**: Build `HashMap<String, usize>` mapping MCP tool names → client index
3. **Tool dispatch**: In ToolUse branch, check MCP dispatch map first. If MCP tool, call `client.call_tool(name, args)`, extract text content
4. **Lifecycle hooks**: on_tool_call fires for MCP tools too

### Task 2: Interpreter exec_agent_loop MCP support
File: `crates/tl-interpreter/src/lib.rs`

Mirror Task 1 for interpreter path.

### Task 3: VM handle_agent_exec — resolve mcp_servers
File: `crates/tl-compiler/src/vm.rs`

In `handle_agent_exec`, when d-field != 0xFF, read mcp_servers list from stack, extract McpClient values, pass to `exec_agent_loop`.

### Task 4: Interpreter exec_agent — resolve mcp_servers
File: `crates/tl-interpreter/src/lib.rs`

Evaluate mcp_servers expressions, extract McpClient values, pass to exec_agent_loop.
